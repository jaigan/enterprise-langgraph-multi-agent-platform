\# Changelog

All notable changes to this project will be documented in this file.

The format follows **Keep a Changelog** and the project adheres to **Semantic Versioning (SemVer)**.

\---

\## Versioning

This project uses Semantic Versioning.

\`\`\`

MAJOR.MINOR.PATCH

\`\`\`

Examples:

\- **MAJOR** → Breaking changes

\- **MINOR** → New backward-compatible features

\- **PATCH** → Bug fixes and documentation updates

Example:

\`\`\`

v1.0.0

\`\`\`

\# Contribution Guidelines

When making changes:

\- Update the **Unreleased** section.

\- Move completed changes into the next version during release.

\- Keep entries concise and descriptive.

\- Group related changes under the appropriate heading.

\- Reference issue or pull request numbers where applicable.

Example:

\`\`\`

\## Added

\- Added support for Anthropic provider (#42)

\## Fixed

\- Fixed checkpoint recovery issue (#57)

\`\`\`
