\# Code of Conduct

\## Our Commitment

We are committed to fostering an open, inclusive, professional, and respectful environment for everyone who participates in this project.

Our goal is to build not only high-quality software but also a collaborative engineering community where contributors can learn, share ideas, and grow together.

Regardless of experience level, background, identity, or location, every contributor should feel welcome and respected.

\---

\# Our Values

We encourage contributors to:

\- Be respectful and professional.

\- Welcome constructive feedback.

\- Share knowledge openly.

\- Help others learn.

\- Communicate clearly.

\- Collaborate in good faith.

\- Focus on solving problems together.

\---

\# Expected Behavior

Contributors are expected to:

\- Treat everyone with respect.

\- Be patient with new contributors.

\- Review code constructively.

\- Provide helpful feedback.

\- Respect different technical opinions.

\- Keep discussions focused on improving the project.

\- Follow repository guidelines and documentation.

\---

\# Unacceptable Behavior

The following behaviors are not tolerated:

\- Harassment or discrimination.

\- Personal attacks.

\- Hate speech.

\- Threats or intimidation.

\- Trolling or inflammatory comments.

\- Deliberate misinformation.

\- Publishing private or confidential information.

\- Sharing credentials, secrets, or sensitive data.

\- Disruptive or abusive behavior.

\---

\# Engineering Standards

All contributors should strive to:

\- Write clean, maintainable code.

\- Follow project architecture.

\- Follow coding standards.

\- Write meaningful commit messages.

\- Include tests where appropriate.

\- Update documentation when necessary.

\- Respect security best practices.

\---

\# Code Reviews

Code reviews should:

\- Focus on the code, not the person.

\- Explain suggested improvements.

\- Encourage discussion.

\- Remain constructive.

\- Avoid dismissive or disrespectful language.

Good review comments:

\- "Could we simplify this logic?"

\- "Would dependency injection improve testability?"

\- "Can we add a unit test for this case?"

Avoid comments such as:

\- "This is terrible."

\- "You don't know what you're doing."

\- "Rewrite everything."

\---

\# Communication Guidelines

We encourage:

\- Asking questions.

\- Sharing knowledge.

\- Explaining design decisions.

\- Supporting new contributors.

\- Respecting different viewpoints.

Healthy technical debate is welcome.

Personal attacks are not.

\---

\# Reporting Issues

If you experience unacceptable behavior:

1\. Contact the project maintainers privately.

2\. Provide relevant details.

3\. Include supporting evidence if available.

4\. Allow time for investigation.

Reports will be handled confidentially whenever possible.

\---

\# Enforcement

Project maintainers may take appropriate action, including:

\- Private warning.

\- Request for behavior changes.

\- Removal of inappropriate content.

\- Temporary suspension.

\- Permanent removal from the project.

Enforcement decisions will be based on the severity and impact of the behavior.

\---

\# Scope

This Code of Conduct applies to:

\- GitHub Issues

\- Pull Requests

\- Discussions

\- Project documentation

\- Community interactions related to the repository

\---

\# Acknowledgements

This Code of Conduct is inspired by the principles of the Contributor Covenant and adapted for an engineering-focused open-source project.
