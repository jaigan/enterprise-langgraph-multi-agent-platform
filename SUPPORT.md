\# Getting Help

If you need assistance, we recommend following this order:

1\. Read the documentation.

2\. Check the FAQ.

3\. Review existing GitHub Issues.

4\. Search previous Discussions.

5\. Create a new Issue if necessary.

Many common questions are already answered in the project documentation.

\---

\# Before Asking for Help

Please gather the following information before creating an issue.

\## Environment

\- Operating System

\- Python Version

\- Package Version

\- LangGraph Version

\- LangChain Version

\---

\## Deployment Environment

Examples:

\- Local Development

\- Docker

\- Kubernetes

\- AWS

\- Azure

\- GCP

\---

\## Problem Description

Please include:

\- Expected behavior

\- Actual behavior

\- Error messages

\- Stack trace

\- Steps to reproduce

\# Reporting Bugs

When reporting bugs, include:

\## Summary

Short description of the issue.

\---

\## Environment

Operating system

Python version

Deployment platform

\---

\## Steps to Reproduce

Example:

1\. Start application

2\. Call API

3\. Execute workflow

4\. Observe failure

\---

\## Expected Result

Describe what should happen.

\---

\## Actual Result

Describe what actually happened.

\---

\## Logs

Include relevant logs.

Never include:

\- Passwords

\- API Keys

\- Tokens

\- Secrets

\---

\# Requesting Features

Feature requests should explain:

\- Business problem

\- Proposed solution

\- Alternative approaches

\- Expected benefits

Good feature requests focus on solving real problems rather than specific implementations.

\- LangGraph

\# Response Expectations

Support is provided on a best-effort basis.

Response times depend on:

\- Issue severity

\- Project priorities

\- Contributor availability

Critical security issues receive the highest priority.
